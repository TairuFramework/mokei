# MCP 2025-11-25 Conformance Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the MUST-level and schema-shape non-conformances against the `2025-11-25` MCP revision that mokei already targets.

**Architecture:** Three sequential phases by package-dependency order â€” (A) `context-protocol` schemas/constants, (B) `context-client`/`context-server` capability + behavior, (C) `http-client`/`http-server` transport MUSTs. Cross-package vitest resolves built `lib/`, not `src/`, so each phase rebuilds `lib/` before the next phase's tests run. Schema items are driven by a reusable conformance-fixture harness (valid/invalid samples validated against schemas); behavior items use focused per-package TDD.

**Tech Stack:** TypeScript, `@enkaku/schema` (JSON Schema validation, `createValidator`), vitest, pnpm workspace.

**Spec:** `docs/superpowers/specs/2026-06-16-mcp-2025-11-25-conformance-design.md`

## Conventions (read once before starting)

- **Type rules (from AGENTS.md):** `type` not `interface`; `Array<T>` not `T[]`; no `any`; uppercase abbreviations (`ID`, `HTTP`, `URL`); no `npm`/`npx`.
- **Validators:** `const v = createValidator(schema)`. `v(value)` returns `{ issues }` â€” `issues == null` means valid, `issues != null` means invalid.
- **Build:** `pnpm build` (full) or per-package `pnpm --filter @mokei/<pkg> build`. After editing a package's `src`, rebuild its `lib/` before testing a dependent package.
- **Test a package:** `pnpm --filter @mokei/<pkg> test`. Narrow to one test: append `-- -t "test name substring"`.
- **Lint:** `rtk proxy pnpm run lint` (NOT `pnpm lint`).
- **Test files:** one `test/lib.test.ts` per package (plus named files like `transport.test.ts`). New fixture harness lives in `packages/context-protocol/test/conformance/`.
- **Commit:** small, per task. Conventional Commits. End body with the Co-Authored-By trailer used in this repo.

Create `packages/context-protocol/test/conformance/harness.ts`:

```typescript
import { createValidator, type Schema } from '@enkaku/schema'
import { expect, test } from 'vitest'

export type ConformanceCase = {
  name: string
  message: unknown
  valid: boolean
}

export type ConformanceFixture = {
  /** Human label for the SEP / feature under test. */
  sep: string
  cases: Array<ConformanceCase>
}

/**
 * Run every case in a fixture against a schema, asserting accept/reject.
 * A `valid: true` case must validate with no issues; `valid: false` must produce issues.
 */
export function runConformance(schema: Schema, fixture: ConformanceFixture): void {
  const validate = createValidator(schema)
  for (const testCase of fixture.cases) {
    test(`${fixture.sep}: ${testCase.name} (${testCase.valid ? 'accept' : 'reject'})`, () => {
      const result = validate(testCase.message)
      if (testCase.valid) {
        expect(result.issues, JSON.stringify(result.issues)).toBeUndefined()
      } else {
        expect(result.issues).toBeDefined()
      }
    })
  }
}
```

Note: `createValidator` returns `{ issues }` where `issues` is `undefined` when valid. If the installed `@enkaku/schema` uses `null` instead, change `.toBeUndefined()` to `.toBeNull()` and the `valid:false` branch to `expect(result.issues).not.toBeNull()`. Verify against the existing usage in `packages/context-server/src/definitions.ts:35` (`validated.issues == null` means valid).

  const DEFAULT_LOCALHOST_ORIGINS = [
    'http://localhost',
    'http://127.0.0.1',
    'http://[::1]',
    'https://localhost',
    'https://127.0.0.1',
    'https://[::1]',
  ]

  function isLocalhostOrigin(origin: string): boolean {
    // Match scheme+host with any port: e.g. http://localhost:3000.
    return DEFAULT_LOCALHOST_ORIGINS.some(
      (base) => origin === base || origin.startsWith(`${base}:`),
    )
  }

  function validateOrigin(request: Request): boolean {
    // Explicit wildcard opts out entirely.
    if (allowedOrigins?.includes('*')) {
      return true
    }
    const origin = request.headers.get('Origin')
    if (allowedOrigins == null) {
      // Secure default: localhost only. No Origin header (non-browser client) is allowed.
      return origin == null || isLocalhostOrigin(origin)
    }
    // Allowlist configured: a missing Origin is not allowed.
    if (origin == null) {
      return false
    }
    return allowedOrigins.includes(origin)
  }
