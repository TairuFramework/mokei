# MCP 2025-11-25 conformance fixes

**Status:** done — shipped on `fix/mcp-spec-conformance` (see `docs/superpowers/plans/2026-06-16-mcp-2025-11-25-conformance.md` for the execution plan and `roadmap.md` shipped list)
**Origin:** 2026-06-12 full audit (security, stability, usability, MCP-spec).

## Gap

Mokei targets `2025-11-25` (`packages/context-protocol/src/rpc.ts:3`) but has hard
non-conformances against it: MUST-level lifecycle/capability/transport rules and wrong
schema shapes for 2025-11-25 features (sampling tools, SEP-1330 enums). Unlike the
draft breaking-cut (blocked), all of this is implementable now and survives the draft
migration largely intact.

## Scope (MUSTs first)

1. **Client protocolVersion check** — `packages/context-client/src/client.ts:193-197`:
   literal `// TODO: check result`; client accepts any responded `protocolVersion`.
   Spec: MUST disconnect if it cannot support the server's version. Compare against a
   supported set; dispose transport and throw on mismatch.
2. **Declare `logging` + `completions` capabilities** —
   `packages/context-server/src/server.ts:106,128-131,202-217`: server sends
   `notifications/message` and serves `completion/complete` without declaring either
   capability (both MUST declare). Set `logging: {}` always, `completions: {}` when
   `params.complete != null`.
3. **Sampling `toolChoice` shape** — `packages/context-protocol/src/sampling.ts:71-113`:
   uses `{type: 'auto'|'required'|'tool', toolName}`; spec (SEP-1577) is
   `{mode: 'auto'|'required'|'none'}`. Rename key, replace the `tool` variant with `none`.
4. **Sampling content blocks** — `packages/context-protocol/src/sampling.ts:58-68,190-215`:
   `SamplingMessage.content` allows only single `text|image`; missing `audio`,
   `tool_use`/`tool_result`, and the array form — conformant `sampling/createMessage`
   requests are rejected `INVALID_REQUEST` by validation. Also reconcile with
   `initialize.ts:48-61`, which already declares `sampling.tools` capability the schemas
   can't carry (complete SEP-1577 or drop the property).
5. **SEP-1330 titled enums** — `packages/context-protocol/src/schema.ts:80-156`: only
   legacy `enum`+`enumNames`; titled multi-select (`items: {anyOf: [{const,title}]}`)
   from a conformant server fails validation and rejects the whole `elicitation/create`.
   Add `oneOf`/`anyOf` const+title variants. Also `ElicitResult.content`
   (`elicitation.ts:106-115`): add `{type:'number'}` and `{type:'array', items:{type:
   'string'}}` (multi-select results currently inexpressible; non-integer numbers
   rejected). Minor while there: `schema.ts:23-77` — add `pattern` to string schema,
   make min/max `number` not `integer`.
6. **Negotiated-version HTTP header** — `packages/http-client/src/transport.ts:103,282,336`:
   always sends `LATEST_PROTOCOL_VERSION`; MUST send the version negotiated at
   initialization. Feed `InitializeResult.protocolVersion` back into the transport.
7. **404 → re-initialize** — `packages/http-client/src/transport.ts:159-162`: on 404
   with an active `Mcp-Session-Id` the client MUST start a new session with a new
   `initialize`; mokei just throws. Clear `#sessionID` and re-initialize (or surface a
   typed signal the client layer acts on).
8. **Origin validation default** — `packages/http-server/src/handler.ts:112-122`:
   validation skipped entirely when `allowedOrigins` is unset (the default) — spec MUST
   (DNS-rebinding protection; 2025-11-25 specifies 403, which the existing reject path
   already returns). Default to localhost-only origins or require explicit opt-out.
   Related (low): a request with *no* Origin header passes even when an allowlist is
   configured (`handler.ts:116-119`) — treat missing Origin as not-allowed when
   configured. Cross-ref: `next/2026-06-12-monitor-daemon-security.md` covers the
   monitor's separate Origin problem.
9. **SEP-1699 cross-stream replay** — `packages/http-server/src/handler.ts:331-342` +
   `sse-writer.ts:69-76`: resumption must work via GET for events from *any* stream;
   mokei replays only the previous GET stream's in-memory buffer, so per-request
   POST-stream events are never replayable. Index replay buffers per session across all
   streams, resolving `Last-Event-ID` via the encoded stream identity.
10. **Tool errors as results** — `packages/context-server/src/server.ts:249-259` +
    `context-rpc/src/rpc.ts:170-175`, `definitions.ts:62-64`: handler exceptions become
    JSON-RPC errors and `createTool` throws `INVALID_PARAMS` on bad input; SHOULD be
    `{isError: true}` tool results (SEP-1303 includes input-validation errors). Catch in
    `#callTool` and convert.
11. **URL elicitation completion** — add `notifications/elicitation/complete` schema and
    `URL_ELICITATION_REQUIRED = -32042` constant (`context-protocol/src/server.ts:34-44`,
    `rpc.ts:6-10`). The request schema already exists; the completion path doesn't.
12. **Misc small** — add `-32002` resource-not-found constant (`rpc.ts:6-10`; draft
    moves it to `-32602`, note in breaking-cut); pass `_meta.progressToken` + a progress
    emitter into server handler context (currently no way to emit
    `notifications/progress`); declare `listChanged: true` where `notify('tools/
    list_changed')` etc. is exposed (`server.ts:137-154`); validate the
    `MCP-Protocol-Version` header server-side (400 on unsupported, assume `2025-03-26`
    if absent); client-side capability gating before `tools/list`/`setLoggingLevel`/
    `complete` (`host.ts:386-396`, `client.ts:246-252`) and METHOD_NOT_FOUND for
    `roots/list` when the capability wasn't declared (`client.ts:221-229`).
