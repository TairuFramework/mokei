## Key design decisions (preserved from spec)

- **Supported versions strict:** single exported `SUPPORTED_PROTOCOL_VERSIONS =
  ['2025-11-25']`; every check reads it. Widening later is one line.
- **Absent `MCP-Protocol-Version` header ŌåÆ allowed** (back-compat, treat as the
  version negotiated at initialize); present-but-unsupported ŌåÆ 400. Applied
  uniformly to POST/GET/DELETE.
- **Origin validation secure-by-default:** unset `allowedOrigins` ŌćÆ localhost-only
  (`http://localhost`/`127.0.0.1`/`[::1]`, any port, http+https); `'*'` opts out;
  missing Origin with an allowlist configured ŌćÆ 403. `:`-separator host match closes
  the suffix-domain bypass. **Behavior change for current users ŌĆö note in changelog.**
- **404 ŌåÆ typed signal, not silent auto-reinit:** transport clears `#sessionID` and
  surfaces `SessionExpiredError`; the client layer owns the re-`initialize` decision.
  Retry policy on top is owned by the separate http-transport-resilience backlog.
- **Tool errors as results:** only tool-execution exceptions and `createTool`
  input-validation (`INVALID_PARAMS`, SEP-1303) convert to `isError` results;
  protocol/transport errors (parse, method-not-found) stay JSON-RPC errors.
- **Lazy-init gating:** client capability gate must `await initialize()` before the
  synchronous capability check ŌĆö otherwise the gate fires against empty
  `#serverCapabilities` (surfaced as a real host-test breakage during execution).
- **Cross-stream replay correctness:** a session-level ordered replay index records
  genuine serverŌåÆclient events only ŌĆö priming events and re-replayed events are
  excluded, and replays preserve their original event ids (via a raw-write path) so
  the client's resumption cursor stays consistent. `eventsAfter` falls back to the
  full buffered log on an unknown/trimmed `Last-Event-ID` (best-effort, no silent
  data loss).
